## IIIC Authority Contract

IIIC implementation must obey these rules:

- Plate-local duplicated triangulations remain crust authority.
- Global TDS samples remain projection and resampling targets, not persistent tectonic authority.
- No ownership recovery, repair, backfill, retention, hysteresis, anchoring, or projection-history hints.
- No centroid or random tie-break in any IIIC primary claim. Existing policies remain comparison controls only.
- Subducting-triangle marks are plate-local/per-window process state and reset at remesh.
- Projection may read subducting/colliding filters, but projection output cannot become the source of carrier authority.
- Vertex positions remain on the unit sphere through Phase III. Elevation is a scalar field only until Phase IV.
- Phase III observability maps are read-only debug artifacts. They do not substitute for numerical gates.

Slab pull is not part of IIIC.1, but it is the first planned place where process output modifies carrier motion authority. It gets a separate gate before IIIC.4:

| Failure mode | Symptom | Required distinguishing evidence |

| Formula self-agreement | Test compares implementation output to implementation output | Independent oracle recomputes expected axis update from thesis/config constants and fixture geometry. |

## Required IIIC Gates

These gates apply before Phase IIIC can advance beyond each slice checkpoint:

| Gate | Requirement |
|---|---|
| Bypass gate | With IIIC disabled, projection/state/crust/material/convergence hashes match the IIIB-equivalent baseline byte-for-byte. |
| Locality gate | Subducting marks, elevation mutation, uplift, and later slab-pull inputs are plate-local/per-window and reset at remesh. |
| Independent oracle gate | IIIC expected values are computed from fixture geometry and thesis/config formulas, not from implementation-reported counters. |
| Ledger gate | Any material or elevation delta has a named record and reconciles to floating-point tolerance. |
| Report-claim gate | IIIC reports may claim only subduction/elevation/slab-pull behavior. They may not claim Stage 1.5 carrier success or Slice 5.5 asymmetry resolution. |
| Stop gate | If IIIC improves projection metrics by weakening authority conservation or depending on projection history, pause for investigation. |

Formula-specific oracle details are intentionally per-slice: IIIC.1 has mark/locality oracles only, while IIIC.3 must restate the uplift formula oracle and IIIC.4 must restate the slab-pull rotation-axis oracle before either slice can land.
