## Active Naming Rule

New work uses the CarrierLab Milestone structure only. Historical names such as
V2, Stage 1.5, Phase IIIB, and Phase IIIE may appear in preserved reports and
source filenames, but new goals, entry packets, checkpoint reports, and user
go/no-go decisions should use one milestone name at a time.

## Advancement Rule

Every milestone must have:

- one question;
- one allowed scope;
- one forbidden scope;
- one numeric evidence gate;
- one checkpoint report;
- explicit user go/no-go before the next milestone.

Visuals and editor actors are diagnostic surfaces. They may explain evidence,
but they do not replace numeric gates or source-grounded checkpoint reports.
